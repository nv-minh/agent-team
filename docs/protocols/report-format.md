# Agent Report Format Protocol

## Overview

This document defines the standard report format for all agents in the EM-Skill distributed orchestration system. Standardized reports enable the Tech Lead to consolidate findings from multiple agents.

## Report Structure

All agent reports must follow this structure:

```markdown

**Report ID:** [Unique ID]

**Agent:** [Agent Name]
**Task ID:** [Task ID]
```

## Report Validation

All reports must pass validation:

### Required Fields

```yaml
required_fields:
  - report_id
  - generated
  - agent
  - task_id
  - status
  - executive_summary
  - findings
  - recommendations
  - scorecard
  - conclusion
```

### Validation Rules

```yaml
validation_rules:
  report_id:
    pattern: "^RPT-[0-9]{4}-[0-9]{3}$"

  generated:
    format: "ISO 8601 timestamp"

  status:
    enum: ["completed", "partial", "failed"]

  overall_status:
    enum: ["pass", "warning", "fail"]

  findings:
    must_include: ["critical", "high", "medium", "low"]
    at_least_one: true

  scorecard:
    overall:
      min: 1
      max: 10

  blocking:
    type: boolean
```

## Best Practices

### Writing Effective Reports

1. **Be Specific:** Use concrete examples and file paths
2. **Be Actionable:** Every finding should have a clear fix
3. **Be Concise:** Executive summary should be 2-3 sentences
4. **Be Quantified:** Use metrics where possible (e.g., "2.3s" not "slow")
5. **Be Prioritized:** Always include severity levels

### Report Quality Checklist

- [ ] Report ID included and unique
- [ ] All required sections present
- [ ] Findings categorized by severity
- [ ] Every issue has a fix
- [ ] Recommendations prioritized (immediate/short/long term)
- [ ] Scorecard completed
- [ ] Handoff section included (if applicable)
- [ ] Artifacts listed (if any)
- [ ] Clear conclusion with blocking issues
