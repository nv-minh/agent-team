## 📝 Contributing to Documentation

When contributing documentation:

1. **Follow the template** - Use existing docs as templates
2. **Be clear and concise** - Write for your audience
3. **Include examples** - Show, don't just tell
4. **Keep it updated** - Documentation should match code
5. **Use consistent formatting** - Follow Markdown best practices

### Documentation Guidelines

- **Language:** Write in clear, simple English
- **Tone:** Professional but approachable
- **Structure:** Use headings, lists, and code blocks
- **Examples:** Include real-world examples
- **Links:** Cross-reference related docs
- **Diagrams:** Use ASCII art for diagrams
- **Version:** Document version-specific changes
