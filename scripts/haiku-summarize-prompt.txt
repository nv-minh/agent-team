# Information Preservation Rules

## 100% Preserve (Must Keep Completely)

### YAML Frontmatter
- `report_id` - Unique identifier
- `agent` - Agent name/role
- `task_id` - Task identifier
- `status` - Overall status (completed/pending/failed)
- `overall` - Overall assessment (pass/fail/warn)
- `confidence` - Confidence score
- All timestamps and metadata

### Critical Data Tables
- **Critical Issues** table (all columns, all rows)
- **High Issues** table (all columns, all rows)
- **Blocking Issues** list (all items)

### Scorecard Data
- Overall score
- Pass/fail counts
- Critical metrics

### Decision Flags
- `ready_to_proceed` - Boolean flag
- `requires_human_review` - Boolean flag
- `blocking_issues` - List of blockers

## 80% Preserve (Condense to Key Points)

### Executive Summary
- Reduce to 2-3 sentences capturing main outcome
- Keep overall status and key findings
- Remove verbose explanations

### Medium Issues
- Show counts only (e.g., "5 medium issues found")
- Group by category if possible
- Remove individual issue details

### Immediate Action Recommendations
- Keep as bullet points
- Condense each action to 1 sentence
- Prioritize by importance (high/medium/low)

### Key Findings
- Summarize to 3-5 bullet points
- Focus on actionable insights
- Remove supporting evidence and details

### Agent Handoff Context
- Keep brief summary of what was done
- Remove detailed step-by-step logs
- Focus on deliverables and outcomes

## 30% Preserve (Heavy Summarization)

### Detailed Analysis Sections
- Summarize to 1-2 paragraphs per section
- Focus on conclusions, not process
- Remove examples and elaboration

### Code Examples
- Keep only critical snippets (max 5-10 lines each)
- Remove full implementations
- Focus on patterns/concepts

### Verbose Explanations
- Reduce to single sentence per point
- Remove redundancy
- Focus on core message

### Low Priority Issues
- Summarize to counts only
- Remove individual details
- Group by category

### Long Descriptions
- Condense to 2-3 sentences
- Keep actionable information
- Remove background context

### Artifacts and Attachments
- Mention existence only
- Remove detailed content
- Provide file paths/names only

1. **Maintain Structure**: Keep the same section organization as original
2. **Preserve YAML**: Keep YAML frontmatter exactly as-is (system will add metadata)
3. **Use Markdown**: Maintain markdown formatting for readability
4. **Be Concise**: Remove all fluff, filler, and redundancy
5. **Stay Factual**: Do not add information not present in original
6. **Preserve Tone**: Maintain professional/technical tone
