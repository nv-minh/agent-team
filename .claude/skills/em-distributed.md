# EM:Distributed - Distributed Investigation

Parallel investigation across multiple specialist agents.

## Usage

```
Use the em:distributed skill to investigate [complex issue]
```

## Examples

```
Use the em:distributed skill to investigate the full-stack performance issue
Use the em:distributed skill to investigate the payment system failure
Use the em:distributed skill to investigate the cross-service data inconsistency
Use the em:distributed skill to investigate the production incident
```

## What This Does

Invokes the **em-distributed-investigation workflow** to:
- Spawn specialist agents in parallel
- Investigate across backend, frontend, database
- Collect and consolidate findings
- Generate comprehensive report
- Coordinate across team members

## Parallel Agents

- **em:backend** - Backend investigation
- **em:frontend** - Frontend investigation
- **em:database** - Database investigation
- **em:debug** - Root cause analysis
- **em:security** - Security assessment
